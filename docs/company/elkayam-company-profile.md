- שילוט חניונים ושילוט חניה
- שילוט אינודייז חריטה (אנודייז + חריטה)
- שילוט נתיבי ישראל / תחבורה ציבורית
- שילוט לתחנות דלק פז
- שילוט מיוחד (אירועים, תשתיות, קמפיינים)

- שלטי נגישות (שירותים, חניה מוגנת, עמדת נגישה)

- שלטי כוונה ומילוט / מרחב מוגן (מקלט, יציאת חרום)
- שלטי קומות לחניון
- שלטי LED סולאריים / חשמליים
- אותיות תלת מימד
- שלט לפי מידה (ייצור לפי הזמנה)

## 8. לוגיקה תפעולית

### כיצד מתחברים רכיבי המערכת
```
לקוח → הזמנת עבודה (Order Form)
  ├── תמרורים (SignRow) → מחסן + ייצור + שטח
  ├── שלט לפי מידה (MiscRow) → ייצור + שטח
  ├── אביזרים (MiscRow) → מחסן + שטח
  └── מוצרים ושירותים נוספים (MiscRow) → שטח + ייצור + לוגיסטיקה

הזמנת עבודה → יומן עבודת שטח (Field Work Log)
  └── צוות מדווח על ביצוע בפועל

קטלוג → מקור אמת לכל הפריטים
  ├── product — מוצר פיזי מהמחסן
  ├── service — שירות/עבודה
  ├── labor — כוח אדם
  └── equipment — ציוד/מכונה
```

### מה שייך לאיזה חלק
- **תמרורים** — כל תמרור ממוספר לפי לוח התמרורים הישראלי (101–935)
- **שלט לפי מידה** — שלט מותאם: ממדים, חומר, עיצוב
- **אביזרים** — מוצרים פיזיים מוכנים מהמחסן (קונוסים, גדרות, פסי האטה, שלטי בטיחות, מחסומים, מראות)
- **מוצרים ושירותים נוספים** — עבודות שטח, שירותים, שירותי ייצור (CNC, UV, אפוקסי, הסדרי תנועה)

## 9. שימוש עתידי — agent logic / system usage

### קטלוג
- כל פריט חדש ייווצר עם `default_price: NULL` (לפי הצעת מחיר)
- סוג: product / service / labor / equipment
- קטגוריה: לפי הטקסונומיה בסעיף 4

### Order Form
- חיפוש autocomplete ב"אביזרים" → `type IN (product, material, equipment)`
- חיפוש autocomplete ב"מוצרים ושירותים נוספים" → `type IN (service, labor, misc, equipment)`
- שלטי רחובות / בטיחות / נגישות → "אביזרים"
- שלטי תדמית / מיוחד / נתיבי ישראל → "שלט לפי מידה"

### Field Work Log
- מחובר להזמנות ומשקף ביצוע בשטח
- אינו דורש שינוי עם הרחבת הקטלוג

### Agent Logic
- כשסוכן מחפש "שלט" — יחפש בקטגוריה "שלטים ושילוט"
- כשסוכן מחפש "הסדר תנועה" — יחפש בקטגוריות "הסדרי תנועה" + "עבודות שטח"
- מינוח עדיף: עברית כמו שמופיע בקטלוג ובאתר הרשמי
